# Claude Code Routines Just Changed Everything

## Executive Summary

The video introduces Claude Code Routines, a new feature allowing users to configure automated prompts that run on schedules or trigger events to audit and improve codebases without manual intervention. The presenter demonstrates two practical applications: an 'auto-improver' that identifies UX gaps and creates pull requests, and a critical security audit routine based on the OWASP Top 10 that automatically fixes vulnerabilities like hardcoded API keys. By running these routines in the cloud, developers can maintain secure and optimized applications continuously while retaining control through pull request reviews or automatic merging.

## Key Points

- Configure Claude Code Routines to run on specific schedules or GitHub events, enabling automated code improvements and security audits that execute in the cloud without requiring your local machine to be active [00:01:42](https://www.youtube.com/watch?v=bzYheNpYl8Y&t=102).
- Implement a security audit routine using a custom skill based on the OWASP Top 10 to automatically detect critical vulnerabilities like hardcoded API keys and generate pull requests with fixes [00:08:05](https://www.youtube.com/watch?v=bzYheNpYl8Y&t=485).
- Ensure custom skills are installed directly within the project repository folder, as standard chat interface skills are not compatible with the automated routine environment [00:09:30](https://www.youtube.com/watch?v=bzYheNpYl8Y&t=570).
- Set up an 'auto-improver' routine to continuously scan your codebase for meaningful UX or feature enhancements, creating isolated feature branches for safe review before merging [00:04:33](https://www.youtube.com/watch?v=bzYheNpYl8Y&t=273).
