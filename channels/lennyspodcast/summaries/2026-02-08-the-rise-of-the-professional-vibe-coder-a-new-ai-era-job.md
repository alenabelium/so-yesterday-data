# The rise of the professional vibe coder (a new AI-era job)

## Executive Summary

Lazar Yavanovich, the first professional vibe coding engineer at Lovable, explains how he builds production-quality internal and external products using AI tools without any traditional coding background. He argues that coding is now a solved problem and the real skills to optimize for are clarity, taste, judgment, and good design. Lazar shares a detailed workflow involving parallel prototyping, extensive PRD documentation, and deliberate context management to overcome LLM token limitations. The conversation reveals a fascinating glimpse into an emerging career path where non-technical builders leverage AI tools to ship products at unprecedented speed, suggesting that traditional engineering, design, and PM roles are converging into a single "builder" function.

## Key Points

- Lazar spends 80% of his time in planning and chat mode and only 20% executing, emphasizing that clarity of intent and good judgment matter far more than raw coding speed; he recommends starting five parallel projects simultaneously to refine design direction before committing to one. [09:34](https://www.youtube.com/watch?v=0XNkUdzxiZI&t=574)
- To combat LLM context window limitations, Lazar creates a suite of markdown PRD files (master plan, implementation plan, design guidelines, user journeys, and tasks.md) that serve as dynamic reference documents the AI agent reads before each task, effectively outsourcing context management to the tool itself. [31:03](https://www.youtube.com/watch?v=0XNkUdzxiZI&t=1863)
- When debugging, Lazar follows a "4x4 framework": first let the tool self-fix, then add console logs for awareness, then use an external tool like Codex for diagnosis, and finally revert and re-prompt with better clarity -- after each fix, he asks the AI how to prompt it better next time and records learnings into rules.md. [65:32](https://www.youtube.com/watch?v=0XNkUdzxiZI&t=3932)
