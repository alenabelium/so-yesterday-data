# Head of Claude Code: What happens after coding is solved 

## Executive Summary

Boris Churnney, head of Claude Code at Anthropic, reflects on the one-year anniversary of Claude Code, revealing that 100% of his code is now written by AI, he ships 10-30 PRs daily, and has not edited a single line by hand since November. He declares coding "largely solved" and describes the next frontier: Claude coming up with its own ideas by analyzing feedback channels, bug reports, and telemetry, moving toward a co-worker rather than just a coding tool. Boris draws a profound analogy between AI coding and the printing press, suggesting this is a democratization moment that will enable everyone to program, with the product boundary expanding from engineers to all knowledge workers through Co-work.

## Key Points

- Claude Code's growth continues to accelerate and 4% of all GitHub commits are now authored by Claude Code (with private repos likely much higher); Boris traces the product's origin from a two-like internal post to a billion-dollar business, crediting the principle of "latent demand" -- building where users are already hacking around existing tools. [05:35](https://www.youtube.com/watch?v=We7BZVKbCVw&t=335)
- Boris advises builders to bet on the most general model (the "bitter lesson"), avoid over-scaffolding with workflows and fine-tuning, give engineers unlimited tokens to experiment, and build for the model six months out rather than today's capabilities -- noting that scaffolding gains of 10-20% typically get wiped out by the next model release. [04:37](https://www.youtube.com/watch?v=We7BZVKbCVw&t=3877)
- Co-work was built in 10 days entirely with Claude Code and was immediately a bigger hit than Claude Code's initial launch; Boris uses it for everything from paying parking tickets to project management, and recommends starting with simple tool-use tasks before connecting multiple tools and running parallel sessions. [51:55](https://www.youtube.com/watch?v=We7BZVKbCVw&t=3115)
