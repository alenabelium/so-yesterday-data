# The Agent Race Stopped Being About Models. It's About Data.

## Executive Summary

The agent landscape has become overwhelming, with multiple major launches every week from OpenAI, Anthropic, Salesforce, Perplexity, and Moonshot. Nate argues that the real conversation has shifted from model quality to infrastructure -- the launches that matter are the ones that change what your existing tools can reach and how easily you can stack systems together. He introduces a five-question filter for evaluating agent launches and runs five current releases through it: ChatGPT Workspace Agents, Salesforce Headless 360, Microsoft Copilot Wave 3, Kimi K2.6, and Perplexity Personal Computer. The key insight is that companies should stop asking "which agent should I switch to?" and start asking "which wrapper around which model fits this specific job?" -- it's a layering question, not a switching question.

## Key Points

- Salesforce Headless 360 scores highest on the five-question filter because it exposes every major platform capability as an API, MCP tool, or CLI command, letting any agent framework -- Claude Code, Cursor, Codex -- act directly on CRM data without human clicking through the interface. [06:42](https://www.youtube.com/watch?v=dQK_pTXrGDk&t=402)
- Claude is no longer just a standalone product -- it now shows up in three forms: direct Claude (model as product), embedded Claude (inside Microsoft Copilot, Salesforce AgentForce, Perplexity), and managed Claude infrastructure (Anthropic's new managed agents layer for teams building their own agent systems). [17:56](https://www.youtube.com/watch?v=dQK_pTXrGDk&t=1076)
- The five-question filter for evaluating any agent launch: Does it plug into tools you already use? Does it let other agents build on top? Does it own or access data you care about? Is there a growing ecosystem? Can you stack agents on top? Most launches fail these tests, which is exactly the point -- filter for infrastructure over features. [02:19](https://www.youtube.com/watch?v=dQK_pTXrGDk&t=139)
