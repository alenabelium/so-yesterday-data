# Clawdbot to Moltbot to OpenClaw: The 72 Hours That Broke Everything (The Full Breakdown)

## Executive Summary

OpenClaw (formerly Claudebot, then Moltbot) is a local-first AI agent that connects to messaging platforms and can autonomously manage email, calendar, browser, files, and more on your hardware. Its explosive growth to 80,000+ GitHub stars was accompanied by 72 hours of chaos: Anthropic's trademark dispute forced a name change, crypto scammers hijacked the old accounts in a 10-second window, and security researchers found hundreds of exposed instances with open API keys and credentials. The project represents a fundamental tension in agentic AI: the broad permissions that make it useful are exactly what make it dangerous, and the security model for local autonomous agents remains unsolved.

## Key Points

- Security researchers found critical vulnerabilities including default trust of localhost connections, zero-moderation plugin marketplace, and successful prompt injection attacks via email that exfiltrated private keys in under 5 minutes -- the architecture requires punching holes through every security boundary built over 20 years [08:14](https://www.youtube.com/watch?v=p9acrso71KU&t=494)
- Despite the risks, OpenClaw delivers on promises big tech assistants never fulfilled: restaurant reservations via AI-generated phone calls, overnight autonomous coding sessions, and self-improving skill creation -- the demand reveals how timid Siri, Alexa, and Google Assistant have been [14:41](https://www.youtube.com/watch?v=p9acrso71KU&t=881)
- The DRAM price surge (172% since early 2025) and memory reallocation to AI data centers means the window for affordable local AI computing may be narrowing, as consumer hardware competes with hyperscalers for the same scarce silicon [14:41](https://www.youtube.com/watch?v=p9acrso71KU&t=881)
