# I Found 5 Things Agents Need From Your Tools. Most Don't Have Them.

## Executive Summary

The video argues that while the human-centric user interface of issue trackers is becoming obsolete, their underlying data structures are becoming the critical infrastructure for AI agents. Tools like Jira and Linear provide the durable state, ownership, permissions, and audit history that autonomous agents require to coordinate work safely, effectively replacing the need for new, greenfield agent platforms. Consequently, the strategic value of enterprise software has shifted from having flashy AI features to possessing clean, structured data models that agents can reliably read and modify.

## Key Points

- Evaluate your software stack by checking if it possesses durable state, explicit ownership, defined verbs, and queryable history, as these are the five essential components agents need to operate safely [22:58](https://www.youtube.com/watch?v=FDkvRl1RlT0&t=1378).
- Prioritize cleaning up existing workflows and consolidating data in current systems like CRMs and ERPs, because agents struggle significantly with messy operations and implicit tribal knowledge [23:51](https://www.youtube.com/watch?v=FDkvRl1RlT0&t=1431).
- For product builders, focus on exposing clean data models and structural verbs via APIs or MCP servers rather than adding chat interfaces, to ensure your tool becomes part of the agent stack rather than just a wrapper [23:51](https://www.youtube.com/watch?v=FDkvRl1RlT0&t=1431).
- Recognize that incumbent enterprise tools are repricing as strategic assets because they own the 'substrate' of work records and permissions that new agent platforms must borrow to function [25:09](https://www.youtube.com/watch?v=FDkvRl1RlT0&t=1509)
