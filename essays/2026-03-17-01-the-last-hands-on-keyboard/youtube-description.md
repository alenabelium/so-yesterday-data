The Last Hands on the Keyboard: How Software Development Became Agent Orchestration

Anthropic's own engineers no longer write code from scratch. They supervise models that write it. At StrongDM, three engineers produce 16,000 lines of Rust and 9,500 lines of Go per day -- spending $1,000 on AI tokens instead of writing a single line themselves.

The code is the exhaust. The work is somewhere else entirely.

🏗️ Three layers of developer value -- Implementation (nearly free, absorbed by agents), Architecture (shared with AI but requires human organizational knowledge), and Intent (entirely human -- agents don't have opinions about what matters). All the leverage lives in Layer 3.

🎵 The orchestra conductor model -- one agent writes code, another writes tests, a third reviews security, a fourth updates docs. Four AI labs independently converged on the same pattern: decompose, parallelize, verify, iterate. Your new workday: specification in the morning, delegation with blast radius management, review and course-correction, then maintaining the rules files that are your project's institutional memory.

🔧 Five skills that replace coding -- Decomposition (agent-sized tasks with verification criteria), Constraint Encoding (tacit knowledge into machine-readable rules), Sniff-Checking (calibrated intuition for output quality), Blast Radius Management (color change = trivial, database migration = review the rollback plan), Recovery Design (because agents make mistakes at the speed of tokens).
