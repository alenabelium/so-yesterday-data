# NotebookLM Video Package: Dark Code and the Three Pillars

## Files

| File | Description |
|------|-------------|
| `01-essay.md` | The full essay — primary source for the video |
| `02-dark-code.md` | Knowledge page: Dark code concept definition and framework |
| `03-specification-quality.md` | Knowledge page: Writing precise specs for AI systems |
| `04-vibe-coding.md` | Knowledge page: Natural language development and its risks |
| `05-amazon-dark-code-story.md` | Video summary: NateBJones analysis of Amazon's AI coding failures |
| `notebooklm-prompt.md` | Custom prompt for NotebookLM Video Overview generation |
| `youtube-description.md` | Ready-to-paste YouTube video description |

## Workflow

1. Open [NotebookLM](https://notebooklm.google.com/) and create a new notebook
2. Upload `01-essay.md` through `05-amazon-dark-code-story.md` as sources
3. Go to the Audio/Video Overview section
4. Paste the contents of `notebooklm-prompt.md` as the custom prompt
5. Generate the Video Overview
6. Download the generated video
7. Upload to YouTube using `youtube-description.md` for the description
8. Fill in the `XX:XX` timestamp placeholders after reviewing the video

## Video Details

- **Target length:** 10-12 minutes
- **Tone:** Provocative and urgent — open with the Amazon outage, pivot to the framework
- **Angle:** The Amazon disaster reveals dark code. The three pillars framework (spec + tests + code, any two reconstruct the third). Five assurance levels. The trust problem.
- **Hook:** "Production code nobody reads is already running your favorite services."
